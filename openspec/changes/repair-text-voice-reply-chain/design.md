# 设计

## 数据流

文字：输入框 → `send()` → `conversations` 云函数 → `hy3` → 消息落库与界面回复。

语音：原生录音 → 临时上传 → `asr` 云函数调用腾讯云一句话识别 → 立即删除录音 → 转写文本 → 同一个 `send()` → `conversations` 云函数 → `hy3` → 回复。

## 关键决策

1. `hy3` 是代码、部署清单和成长计划环境的统一默认模型，环境变量仍可显式覆盖。
2. 不使用个人主体无法申请的微信同声传译插件。`asr` 云函数只使用运行角色注入的短期身份；运行角色按最小权限仅允许`asr:SentenceRecognition`，不接收或保存长期云密钥。腾讯云账号AppID由运行环境确定，不写入请求、代码或仓库。
3. 按腾讯云原始错误码分别识别未开通、运行角色无权限、临时凭证异常、额度耗尽、欠费隔离和无效音频；不能把所有`FailedOperation.*`统一显示为“服务未开通”。失败时安全降级为文字输入，不自动购买资源包或开启后付费。
4. 用识别管理器的 `onStop` Promise 等待最终文本，替代固定 200ms 竞态等待。
5. 识别失败、超时或空结果只展示可重试提示，不把失败录音当作已发送消息。

## 回滚

移除 `app.json` 插件声明即可回退；文字云函数修复可独立保留。
