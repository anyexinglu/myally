# 设计

## 数据流

文字：输入框 → `send()` → `conversations` 云函数 → `hy3` → 消息落库与界面回复。

语音：原生录音 → 临时上传 → `asr` 云函数调用腾讯云一句话识别 → 立即删除录音 → 转写文本 → 同一个 `send()` → `conversations` 云函数 → `hy3` → 回复。

## 关键决策

1. `hy3` 是代码、部署清单和成长计划环境的统一默认模型，环境变量仍可显式覆盖。
2. 不使用个人主体无法申请的微信同声传译插件。`asr` 云函数只使用平台注入的短期身份，不接收或保存长期云密钥。
3. 语音识别服务未开通、权限不足或免费额度耗尽时安全降级为文字输入，不自动购买资源包或开启后付费。
3. 用识别管理器的 `onStop` Promise 等待最终文本，替代固定 200ms 竞态等待。
4. 识别失败、超时或空结果只展示可重试提示，不把失败录音当作已发送消息。

## 回滚

移除 `app.json` 插件声明即可回退；文字云函数修复可独立保留。
